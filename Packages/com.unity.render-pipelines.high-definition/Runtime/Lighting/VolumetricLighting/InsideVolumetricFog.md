# 体积雾着色器 (Volumetric Fog) 算法文档

## 概述

这是 HDRP 中体积雾后处理着色器的算法分析文档，基于 SPIRV 反编译代码还原而成。核心思想是：**利用蓝噪声进行空间抖动采样，通过深度一致性检测自适应选择滤波策略，消除边缘鬼影伪影**。

## 核心问题解答

### Q: 能否简洁描述整个算法的核心思路？

**A:** 可以理解为以下三个步骤：

1. **空间抖动采样**: 在每个像素点，根据蓝噪声的偏移到周围像素位置，采样四次深度
2. **深度一致性检测**: 比较4个采样深度值，判断当前像素是否处于深度跳变的边缘
3. **自适应滤波**: 边缘处用深度加权滤波，平坦处用简单平均

### Q: 为什么需要边缘检测？

**A:** 边缘处的采样可能跨越不同深度层。

```
场景: 前景物体 ████  背景物体 ░░░░

采样位置:
    ████░░░░
       ↑
    采样点跨越边缘

不加权: f₀=0.9(近) f₁=0.2(远) f₂=0.9 f₃=0.2
    → result = (0.81+0.04+0.81+0.04)/4 = 0.425
    → 边缘处出现半透明鬼影

加权 (假设sceneDepth靠近前景):
    w₀=10, w₁=0.5, w₂=10, w₃=0.5  (近处权重高)
    → result ≈ 0.77  (更接近正确的前景雾效)
```

## 关键技术详解

### 1. 蓝噪声采样

```glsl
UV = (time × prime + screenPos) × texelSize
```

- 使用质数(1259, 1277)混合时间和屏幕坐标
- 产生时域+空域双重抖动效果
- 每帧采样不同的噪声模式，避免静态噪声图案

### 2. 快速随机数生成 (FastRandomFloat)

```glsl
// 原始代码
float _81 = asfloat(uint(int(asuint(_70)) >> 1u) + 532487669u);

// 原理: IEEE 754 单精度浮点数结构
// ┌─ 符号 (1 bit) ─┬─ 指数 (8 bits) ─┬─ 尾数 (23 bits) ─┐
// │     S          │    EEEEEEEE     │    MMMMMMMMMMM...│
// └────────────────┴─────────────────┴──────────────────┘
//
// 右移1位: 符号位变0，指数位被部分保留，产生 [0,1) 范围的值
// +532487669: 魔法常数，调整指数位使结果分布更均匀
```

### 3. 24位深度解包

```glsl
// 原始代码
float depth = dot(packedDepth.yzw, float3(1.0, 1/255, 1/65535));

// 即: depth = y * 1.0 + z * (1/255) + w * (1/65535)
// 将YZW三个通道解码为线性深度（24位精度）
```

### 4. 深度边缘检测

```glsl
depthContrast = (maxDepth - minDepth) / avgDepth

// 如果 contrast < 0.1: 使用简单平均
// 否则: 使用深度加权平均
```

## 数据流总结

| 阶段 | 输入 | 输出 |
|------|------|------|
| 1. 噪声采样 | screenUV + time | blueNoise (RGBA) |
| 2. 偏移生成 | blueNoise.x, .y | 4个偏移向量 |
| 3. 雾效采样 | 偏移位置 | 4组 (fog, depth) |
| 4. 边缘检测 | 4个depth | contrast值 |
| 5. 滤波选择 | contrast | fogResult |
| 6. 合成 | fogResult + fogColor + dither | 最终颜色 |

---

## 两种滤波策略对比

| | 平坦区域 (contrast < 0.1) | 边缘区域 (contrast ≥ 0.1) |
|---|---|---|
| **场景** | 空旷区域、墙面、天空 | 物体边缘、前景/背景交界 |
| **算法** | 简单平均 | 深度加权平均 |
| **公式** | `(f₀²+f₁²+f₂²+f₃²) / 4` | `Σ(wᵢ×fᵢ²) / Σ(wᵢ)` |
| **目的** | 快速，降噪 | 消除鬼影 |
| **权重** | 各点相等 | 越接近sceneDepth权重越高 |

## 顶点着色器：射线参数计算 (`inside_calculate_volumetric_fog_data_vertShader.glsl`)

### 功能

顶点着色器负责计算光线步进所需的射线参数，包括射线与雾盒的相交位置。这些参数传递给片段着色器用于光线步进。

### 输入输出

**输入：**
- `POSITION0`: 顶点位置（全屏四边形或包围盒）
- `NORMAL0`: 顶点法线（未使用）
- `TEXCOORD0`: 纹理坐标（未使用）

**输出：**
- `SV_POSITION0`: 裁剪空间位置
- `TEXCOORD0` (rayParams): 射线参数 (tNear, 0, 0, 1)
- `TEXCOORD1` (boxStartCoeffs): 盒相交起始系数
- `TEXCOORD2` (boxStartPos): 盒相交起始位置
- `TEXCOORD3` (boxEndCoeffs): 盒相交结束系数
- `TEXCOORD4` (boxEndPos): 盒相交结束位置
- `TEXCOORD5` (volumeParams): 体积参数

### 关键数学概念

**射线与盒相交**

顶点着色器计算的是射线与雾盒的相交参数。射线定义为：

```
R(t) = origin + t × direction
```

其中：
- `origin`: 相机位置（世界空间）
- `direction`: 射线方向（归一化）
- `t`: 沿射线的距离

片段着色器使用这些参数进行光线步进，从 `tNear` 到 `tMax` 采样雾的密度。

**为什么要分两步？**

1. **顶点着色器**: 计算几何参数（射线起点、终点、方向）
2. **片段着色器**: 沿射线步进，累积雾的透射率

这种分离使得每个片段可以独立进行光线步进，而不需要重复计算几何参数。

## Pass 1: 雾密度数据生成片段着色器 (`inside_calculate_volumetric_fog_data_refactored.glsl`)

### 功能

在 **1/4 分辨率** 上，为每个像素计算体积雾的透射率(Transmittance)，并打包深度值到 T1 纹理。

### 输入纹理

| 纹理 | 用途 |
|------|------|
| T0 (BlueNoiseTex) | 蓝噪声，用于抖动 |
| T1 (RampTex) | 横向条 ramp 纹理，1D 光照衰减曲线 |
| T2 (CircularMaskTex) | 圆形渐变 mask 纹理，聚光灯形状遮罩 |
| T3 (ShadowMapTex) | 阴影/遮蔽图，PCF 采样 |
| T4 (LightDepthTex) | 灯光起点到终点的距离纹理（近处值大，远处值小） |
| T5 (CameraDepthTex) | 游戏相机视角的深度纹理 |

### 算法流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    雾密度数据生成流程                            │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  Step 1: 相机深度裁剪                                           │
│  ──────────────────────────                                   │
│  读取 T5 的相机原始深度，T4 的灯光线性化深度                    │
│  如果像素被遮挡 (mad(fadeScale, rawDepth, -linearDepth) < 0)   │
│  则 discard，不计算雾效                                         │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  Step 2: 蓝噪声 + 射线区间计算                                   │
│  ────────────────────────────────────                           │
│                                                                 │
│  从顶点着色器传入的 TEXCOORD 系列参数:                           │
│    TEXCOORD = tNear  (射线近端)                                  │
│    TEXCOORD_1/2 = 射线起点系数/位置 (box entry)                │
│    TEXCOORD_3/4 = 射线终点系数/位置 (box exit)                 │
│                                                                 │
│  射线区间:                                                      │
│    tMin = min(lightDepthLinear, boxEntry)                      │
│    tMax = min(lightDepthLinear, boxExit)                       │
│                                                                 │
│           camera            box exit                             │
│             ↓↗              ↑                                  │
│        ════════●●━━━━━━━━━━━↑ tMax                             │
│              ●  ↑tMin                                          │
│              ●                                              │
│              ●                                              │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  Step 3: 光线步进 (6步, 中点积分)                               │
│  ──────────────────────────────────                             │
│                                                                 │
│  将射线分成6段，每段采样:                                        │
│                                                                 │
│  For each step i:                                              │
│    shadowUV = T4 的 UV 坐标 (灯光深度归一化)                    │
│    shadow   = T3.SampleCmpLevelZero(shadowMap, shadowUV, depth)│
│    mask     = T2.SampleLevel(CircularMask, lightUV, mip)        │
│    ramp     = T1.SampleLevel(Ramp, sampleLength, 0)             │
│                                                                 │
│    fogDensity = shadow * shadowScale + shadowScale               │
│    trans_i = fogDensity * (base2 + ramp * base) * mask          │
│    transmittance += trans_i * Δt                                │
│                                                                 │
│  最终 transmittance = Σ(trans_i × Δt_i) × scale × (1/6)        │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  Step 4: 法线分布校正                                           │
│  ──────────────────────                                         │
│                                                                 │
│  使用 FastRecipSqrt (Carmack 反平方根) 近似:                    │
│    normFactor = FastRecipSqrt(transmittance)                    │
│    normalDist = normFactor * (1.5 - transmittance * normFactor²)│
│                                                                 │
│  等价于: normalDist = 1/sqrt(transmittance * transmittance)    │
│  目的: 调整透射率的分布曲线，更符合物理模型                      │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  Step 5: 输出打包                                               │
│  ─────────────────                                               │
│                                                                 │
│  SV_Target.r = fogValue (透射率经法线分布校正)                   │
│  SV_Target.g = frac(depth * 255)           // bits[8:15]       │
│  SV_Target.b = frac(depth * 65025)          // bits[16:23]      │
│  SV_Target.a = frac(depth * 16777216)       // bits[0:7]       │
│                                                                 │
│  重构: depth = g + b/255 + a/65535                              │
│  解包: depth = y + z/255 + w/65535 (见上方采样 shader)          │
└─────────────────────────────────────────────────────────────────┘
```

### 关键技术: FastRecipSqrt

```glsl
// John Carmack 的快速反平方根
float FastInvSqrt(float x)
{
    float xhalf = 0.5f * x;
    int i = asint(x);
    i = 1597463174 - (i >> 1);  // 魔数 - 右移1位
    x = asfloat(i);
    // 单次牛顿迭代
    x = x * (1.5f - xhalf * x * x);
    return x;
}

// 倒数: 1/sqrt(x) = sqrt(1/x) 的近似
float FastRecipSqrt(float x)
{
    float y = FastInvSqrt(x);
    // 精确化: y = y * (1.5 - 0.5 * x * y * y)
    y = y * (1.5f - 0.5f * x * y * y);
    return y;
}
```

### 24位深度打包验证

```
设 depth = 0.5

scaled255    = 0.5 * 255     = 127.5  → frac = 0.5
scaled65025  = 0.5 * 65025   = 32512.5 → frac = 0.5

y = frac(scaled255) * (1/255)       = 0.5 / 255
z = (-frac65025) + frac255         = -0.5 + 0.5 = 0
w = (-frac65025) + floor(scaled255) = -0.5 + 127  = 126.5

解码: y + z/255 + w/65535 = 0.5/255 + 0 + 126.5/65535 ≈ 0.5 ✓
```

## 用人话解释 Pass 1: 雾数据是怎么算出来的

### 什么是"透射率"(Transmittance)

你可以理解为：**一束光从 A 点穿透到 B 点，有多少比例能穿过去**。

- 透射率 = 1.0：完全没有雾，光 100% 穿透
- 透射率 = 0.0：雾太浓了，光完全穿不过去
- 透射率 = 0.3：只有 30% 的光能穿透，70% 被雾"吃掉"了

### 5步搞懂这个 Shader

---

**第一步：先看看这个像素是不是被墙挡住了**

```
🎯 相机看到的地方

    ╔═══════════╗
    ║           ║
    ║    🧊     ║  ← 墙壁/遮挡物
    ║   这里是  ║
    ║   被挡的  ║
    ╚═══════════╝
```

 Shader 会读取场景深度图，如果一个像素被墙挡住了，就跳过不计算雾效了。这样可以省很多性能。

---

**第二步：沿着视线从近到远画一条"射线"**

```
    相机
      ↓
      ↘
       ↘  ← 射线（从相机出发，穿过像素的方向）
        ↘
         ↘  ← 摄像机能看到的范围是有限的（由雾的边界盒决定）
          ↘
           ↘
```

这条射线会穿过雾的区域。我们只需要计算射线在雾里走过的这段距离。

---

**第三步：把射线分成6段，在每段中间采样**

```
射线从近到远:
[  ●----●----●----●----●----●----●  ]
      ↑   ↑   ↑   ↑   ↑   ↑
    采样  采样  采样  采样  采样  采样
    点1   点2   点3   点4   点5   点6
```

为什么是"中间"而不是均匀分布？因为数学上用中点积分更准确。

**每到一个采样点，Shader 问三个问题：**

1. **这里有光吗？** → 读取阴影图(T3)，看看这个方向有没有被光源照到
2. **光形是怎样的？** → 读取圆形渐变mask(T2)，定义聚光灯的形状范围
3. **光衰减了多少？** → 读取横向ramp曲线(T1)，根据距离查光照衰减

最后把所有采样点的"透光能力"累加起来，就是这个像素的透射率。

---

**第四步：用数学公式让雾看起来更自然**

Shader 会对累加的结果做一个"曲线调整"，让雾的浓淡变化更平滑、更符合物理规律。这就像滤镜一样，让结果更好看。

---

**第五步：把结果和深度打包成一纹理**

```
输出的 T1 纹理每个像素存4个值:

┌─────────────────────────────────────┐
│  R: 透射率（雾浓不浓）              │
│  G: 深度的第9-16位                   │
│  B: 深度的高8位 (第17-24位)          │
│  A: 深度的低8位 (第1-8位)            │
└─────────────────────────────────────┘
```

深度信息会被拆成3个通道存储（每个8位），这样在只占用很少空间的情况下，能保存24位的精度，足够区分很近的两个物体。

### 性能优化：为什么要 1/4 分辨率

```
全分辨率              1/4 分辨率 (我们用的)

████████████████        ████
████████████████        ████
████████████████   →    ████
████████████████        ████
████████████████
████████████████

像素数: 16个           像素数: 4个
计算量: 16次           计算量: 4次
```

1/4 分辨率意味着横向和纵向都只有原来一半的像素，总共只有 **1/16 的计算量**。因为雾是模糊的，低分辨率看起来差别不大，但性能好很多。

## 性能优化要点

1. **1/4 分辨率渲染**：Pass 1 在 1/4 分辨率上计算，计算量减少到 1/16
2. **提前剔除**：被遮挡的像素直接 discard，避免不必要的计算
3. **中点积分**：6 步光线步进使用中点法则，比均匀采样更精确
4. **自适应滤波**：平坦区域用简单平均，边缘区域才用深度加权
5. **蓝噪声抖动**：时域抖动避免静态噪声，空间抖动提高采样质量
